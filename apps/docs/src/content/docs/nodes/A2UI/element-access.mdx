# Element Access Nodes

These nodes allow you to read data from A2UI page elements within workflows.

## Get Element

Retrieves the full element data for an element reference.

### Inputs
| Pin | Type | Description |
|-----|------|-------------|
| Element Ref | String | The element reference (format: `surfaceId/elementId`) |

### Outputs
| Pin | Type | Description |
|-----|------|-------------|
| Element | JSON | Full element data including type, props, style, children |
| Exists | Boolean | Whether the element was found |

### Example Usage

```
Get Element → element_ref: "main-surface/submit-button"
          ↓
       Element → { type: "Button", component: { label: "Submit", ... } }
       Exists → true
```

<Aside type="tip">
Use the ElementSelect dropdown in the flow editor to easily pick elements from your page.
</Aside>

## Get Element Text

Reads text content from elements that display text (Text, Button labels, etc.).

### Inputs
| Pin | Type | Description |
|-----|------|-------------|
| Element Ref | String | Reference to the element |

### Outputs
| Pin | Type | Description |
|-----|------|-------------|
| Text | String | The text content (from `content`, `text`, or `label` property) |
| Has Text | Boolean | Whether text was found |

### Supported Elements
- **Text** - Returns `content` property
- **Button** - Returns `label` or `text` property
- **Input** - Returns `placeholder` as fallback

---

## Get Element Value

Reads the current value from form input elements.

### Inputs
| Pin | Type | Description |
|-----|------|-------------|
| Element Ref | String | Reference to the input element |

### Outputs
| Pin | Type | Description |
|-----|------|-------------|
| Value | String | The current input value |
| Has Value | Boolean | Whether a value exists |

### Supported Elements
- **TextField / Input** - Text input value
- **Select** - Selected option value
- **Checkbox / Switch** - Returns "true" or "false" string
- **Slider** - Numeric value as string

## Type-Specific Getters

### Button Nodes

#### Get Button Label
Returns the label/text displayed on a button.

| Input | Output |
|-------|--------|
| Element Ref (String) | Label (String) |

#### Get Button Disabled
Returns whether the button is disabled.

| Input | Output |
|-------|--------|
| Element Ref (String) | Disabled (Boolean) |

#### Get Button Loading
Returns whether the button shows a loading state.

| Input | Output |
|-------|--------|
| Element Ref (String) | Loading (Boolean) |

---

### Input Nodes

#### Get Input Placeholder
Returns the placeholder text of an input field.

| Input | Output |
|-------|--------|
| Element Ref (String) | Placeholder (String) |

---

### Select Nodes

#### Get Select Value
Returns the currently selected value and whether a selection exists.

| Input | Output |
|-------|--------|
| Element Ref (String) | Value (String), Has Selection (Boolean) |

---

### Checkbox/Switch Nodes

#### Get Checkbox Checked
Returns the checked state of a checkbox or switch.

| Input | Output |
|-------|--------|
| Element Ref (String) | Checked (Boolean) |

## How Element Data is Passed

When a workflow runs from an A2UI page action, all page elements are automatically included in the payload:

```json
{
  "_elements": {
    "surface-id/element-1": {
      "type": "Button",
      "component": { "label": "Click Me", "disabled": false },
      "style": { "className": "btn-primary" }
    },
    "surface-id/element-2": {
      "type": "TextField",
      "component": { "value": "user input", "placeholder": "Enter text" }
    }
  },
  "_route": "/dashboard",
  "_page_id": "/dashboard"
}
```

This eliminates the need for additional API calls to fetch element state.
