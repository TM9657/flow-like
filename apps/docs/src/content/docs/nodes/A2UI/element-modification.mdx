# Element Modification Nodes

These nodes allow you to modify A2UI page elements in real-time from workflows.

## How Updates Work

When you use a setter node:
1. The node sends an update message to the frontend
2. The frontend applies the change immediately
3. The element re-renders with the new state

Updates are streamed in real-time, so users see changes as they happen.

---

## Generic Setter Nodes

### Set Element Text

Updates the text content of text-displaying elements.

| Pin | Type | Description |
|-----|------|-------------|
| ▶ Exec In | Execution | Triggers the update |
| Element Ref | String | Target element reference |
| Text | String | New text content |
| ▶ Exec Out | Execution | Continues after update |

**Supported elements:** Text, Button, any element with `content`, `text`, or `label`

---

### Set Element Value

Updates the value of form input elements.

| Pin | Type | Description |
|-----|------|-------------|
| ▶ Exec In | Execution | Triggers the update |
| Element Ref | String | Target element reference |
| Value | String | New value |
| ▶ Exec Out | Execution | Continues after update |

**Supported elements:** TextField, Select, Checkbox, Switch, Slider

---

### Set Element Style

Merges new style properties into an element's existing style.

| Pin | Type | Description |
|-----|------|-------------|
| ▶ Exec In | Execution | Triggers the update |
| Element Ref | String | Target element reference |
| Style | JSON | Style properties to merge |
| ▶ Exec Out | Execution | Continues after update |

**Example style object:**
```json
{
  "className": "bg-blue-500 text-white",
  "opacity": 1
}
```

---

### Set Element Visibility

Shows or hides an element.

| Pin | Type | Description |
|-----|------|-------------|
| ▶ Exec In | Execution | Triggers the update |
| Element Ref | String | Target element reference |
| Visible | Boolean | `true` to show, `false` to hide |
| ▶ Exec Out | Execution | Continues after update |

<Aside type="note">
Hidden elements have `opacity: 0` and `hidden: true`. They remain in the DOM but are not visible or interactive.
</Aside>

---

### Set Element Disabled

Enables or disables an element (typically buttons and inputs).

| Pin | Type | Description |
|-----|------|-------------|
| ▶ Exec In | Execution | Triggers the update |
| Element Ref | String | Target element reference |
| Disabled | Boolean | `true` to disable, `false` to enable |
| ▶ Exec Out | Execution | Continues after update |

---

### Set Element Loading

Sets the loading state of buttons.

| Pin | Type | Description |
|-----|------|-------------|
| ▶ Exec In | Execution | Triggers the update |
| Element Ref | String | Target element reference |
| Loading | Boolean | `true` to show loading, `false` to hide |
| ▶ Exec Out | Execution | Continues after update |

<Aside type="tip">
Use this at the start of a workflow to show loading, then set to `false` at the end.
</Aside>

## Type-Specific Setter Nodes

### Button Nodes

#### Set Button Label
Updates the button's display text.

| Input | Description |
|-------|-------------|
| Element Ref | Button element reference |
| Label | New label text |

---

### Input Nodes

#### Set Input Placeholder
Updates the placeholder text shown when input is empty.

| Input | Description |
|-------|-------------|
| Element Ref | Input element reference |
| Placeholder | New placeholder text |

#### Clear Input
Clears the input value (sets to empty string).

| Input | Description |
|-------|-------------|
| Element Ref | Input element reference |

---

### Select Nodes

#### Set Select Value
Changes the selected option.

| Input | Description |
|-------|-------------|
| Element Ref | Select element reference |
| Value | Value to select |

---

### Checkbox/Switch Nodes

#### Set Checkbox Checked
Sets the checked state directly.

| Input | Description |
|-------|-------------|
| Element Ref | Checkbox/Switch element reference |
| Checked | `true` or `false` |

#### Toggle Checkbox
Toggles the current state and outputs the new state.

| Input | Output |
|-------|--------|
| Element Ref | New State (Boolean) |

## Common Patterns

### Loading State Pattern

<Steps>
1. **Start of workflow** - Set button loading to `true`
2. **Perform operations** - API calls, data processing, etc.
3. **End of workflow** - Set button loading to `false`
</Steps>

```
[Workflow Start] → [Set Loading: true] → [API Call] → [Set Loading: false] → [End]
```

### Form Validation Pattern

```
[Get Input Value] → [Validate] → [Branch]
                                    ├─ Valid: [Submit]
                                    └─ Invalid: [Set Element Text: "Error message"]
```

### Conditional Visibility Pattern

```
[Get Checkbox Checked] → [Branch]
                            ├─ Checked: [Set Element Visibility: panel → true]
                            └─ Unchecked: [Set Element Visibility: panel → false]
```
