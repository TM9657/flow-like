# A2UI Nodes

A2UI (App-to-UI) nodes allow workflows to interact with page elements - reading their state, modifying them, creating new elements dynamically, querying collections, and managing application state.

## Categories

### Element Access
- **Get Element** - Retrieve element data by reference
- **Get Element Text** - Read text content from elements
- **Get Element Value** - Read input values from form elements

### Element Modification
- **Set Element Text** - Update text content
- **Set Element Value** - Update input values
- **Set Element Style** - Apply style changes
- **Set Element Visibility** - Show/hide elements
- **Set Element Disabled** - Enable/disable elements
- **Set Element Loading** - Control loading states

### Type-Specific Nodes

#### Button Nodes
- **Get Button Label** - Read button text
- **Set Button Label** - Change button text
- **Get Button Disabled** - Check if disabled
- **Get Button Loading** - Check loading state

#### Input Nodes
- **Get Input Placeholder** - Read placeholder text
- **Set Input Placeholder** - Change placeholder
- **Clear Input** - Clear input value

#### Select Nodes
- **Get Select Value** - Read selected value
- **Set Select Value** - Change selection

#### Checkbox/Switch Nodes
- **Get Checkbox Checked** - Read checked state
- **Set Checkbox Checked** - Set checked state
- **Toggle Checkbox** - Toggle and return new state

### Dynamic Elements
- **Create Element** - Create new elements dynamically
- **Remove Element** - Remove elements from page
- **Clone Element** - Duplicate existing elements

### Query Nodes
- **Query Elements by Type** - Find all elements of a type
- **Query Elements by ID** - Find elements matching ID patterns
- **Query Children** - Get children of a container
- **Query Parent** - Get parent of an element

### Navigation
- **Navigate To** - Navigate to a route
- **Get Current Route** - Get current page route
- **Get Route Params** - Get route parameters

### State Management
- **Get Global State** - Read from global state
- **Set Global State** - Write to global state
- **Get Page State** - Read from page-local state
- **Set Page State** - Write to page-local state

## Data Flow

When a workflow is triggered from an A2UI page:

1. All page elements are sent in the workflow payload (`_elements`)
2. Current route is included (`_route`)
3. Route parameters are included (`_route_params`)
4. Current page ID is included (`_page_id`)
5. Global state is included (`_global_state`)
6. Page state is included (`_page_state`)

This allows workflows to access element data without additional round trips.

## Element References

Element references use the format `surfaceId/elementId`. When using getter/setter nodes, provide this full reference.

## State Persistence

- **Global State**: Persisted to IndexedDB, survives page refreshes
- **Page State**: Persisted per-page to IndexedDB, survives navigation

## Best Practices

1. **Use type-specific nodes** when possible (e.g., `Get Button Label` instead of `Get Element` + extracting label)
2. **Batch updates** by connecting multiple setter nodes in sequence
3. **Use query nodes** to operate on collections rather than individual elements
4. **Leverage state management** for cross-page data instead of recreating elements
